2.1 Detail Bug #1: firstWhere tanpa orElse

⚠️ firstWhere() tanpa parameter orElse akan melempar StateError jika elemen tidak ditemukan.

Sebelum (Bermasalah):

final task = _apiController.tasks.firstWhere((t) => t.id == id);
// CRASH jika task tidak ditemukan!

Sesudah (Diperbaiki):

final taskIndex = _apiController.tasks.indexWhere((t) => t.id == id);
if (taskIndex == -1) {
  debugPrint('⚠️ Task with id $id not found');
  return false;
}
final task = _apiController.tasks[taskIndex];

2.2 Detail Bug #3: Race Condition Database

⚠️ Dua pemanggilan database bersamaan bisa menginisialisasi database dua kali.

Solusi: Menggunakan Completer<Database> sebagai lock mechanism.

static Completer<Database>? _completer;

Future<Database> get database async {
  if (_database != null) return _database!;
  if (_completer != null) return _completer!.future; // Tunggu inisialisasi
  _completer = Completer<Database>();
  _database = await _initDatabase();
  _completer!.complete(_database!);
  return _database!;
}
